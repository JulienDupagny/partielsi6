1/ A quoi fait référence le mot clé java this ?
Le mot clé java this fait référence à un objet lui même.

2/ A quoi fait référence le mot clé java super ?
Le mot clé super fait référence à l'objet en cours, ce mot clé permet de désigner une superclasse, il permet enfaite de manipuler les données membres et les méthodes de la superclasse.

3/ Qu'est ce que l'instanciation ?
L'instanciation est l'action d'instancier, de créer un objet à partir d'un modèle.

4/ Quelle est la différence entre un Objet et une Classe ?
Une Classe est un "moule" et l'Objet est fait dans le "moule", cela fait toute la différence !
Une Classe est une recette de gâteau, l'Objet étant UNE réalisation de cette recette.
Ou encore .. Une Classe est une sorte de moule à gâteaux, un Objet est un gâteau que l'on ferais à partir de ce moule.

5/ Qu'est ce qu'un Mock ?
Un Mock est objet simulé qui reproduit le comportement d'objets réels de manière contrôlée. Un programmeur crée un mock dans le but de tester le comportement d'autres objets, réels, mais liés à un objet inaccessible ou non implémenté. Ce dernier est alors remplacé par un mock.

6/ Qu'est ce que le scaffolding ?
Le scaffolding une façon de concevoir des logiciels liés à une base de données. Cette technique est souvent fournie avec le patron de conception Modèle-Vue-Contrôleur, dans lequel le programmeur écrit une spécification décrivant comment la base de données sera utilisée. Le compilateur génère le code source de création, lecture, mise à jour et suppression (CRUD = Create, Read, Update, Delete) des données en base pour l'application.
Le scaffolding est adopté par Symfony (PHP), CakePHP ...

7/ Qu'est ce qu'un RAD ?
C'est l'acronyme de Rapid Application Development qui veut dire en français Développement Rapide d'Applications,
RAD comprend enfaite les logiciels de Développement Rapide d'Applications comme Eclipse, WinDev et autres.

8/ Qu'est ce que le refactoring ?
Le refactoring est enfaite le réusinage de code, cela consiste à retravailler le code source d'un programme informatique.
Le refactoring améliore la lisibilité en identant le code comme il le faut de sorte à le rendre plus générique.
